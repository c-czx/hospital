# 医院挂号系统 - 管理员功能说明

## 功能概述

管理员是系统中权限最高的角色，负责整个系统的管理和维护工作。

## 已实现的管理员功能

### 1. 用户管理

#### 1.1 用户列表
- **路径**: `/admin/users`
- **功能**: 查看所有用户信息，包括：
  - 用户ID
  - 用户名
  - 姓名
  - 角色
  - 性别
  - 电话

#### 1.2 添加用户
- **路径**: `/admin/user/create`
- **功能**: 创建新用户，支持的角色：
  - PATIENT（患者）
  - DOCTOR（医生）
  - NURSE（护士）
  - ADMIN（管理员）

#### 1.3 编辑用户
- **路径**: `/admin/user/edit/{id}`
- **功能**: 修改用户信息，支持：
  - 修改用户名
  - 修改密码（可选）
  - 修改姓名
  - 修改角色
  - 修改性别
  - 修改电话
  - 修改邮箱

#### 1.4 删除用户
- **路径**: `/admin/user/delete/{id}`
- **功能**: 删除指定用户

### 2. 权限管理

#### 2.1 权限列表
- **路径**: `/admin/permissions`
- **功能**: 查看所有用户的权限信息

#### 2.2 更新权限
- **路径**: `/admin/permission/update/{id}`
- **功能**: 修改用户的角色权限

### 3. 科室管理

#### 3.1 科室列表
- **路径**: `/admin/departments`
- **功能**: 查看所有科室信息

#### 3.2 添加科室
- **路径**: `/admin/department/create`
- **功能**: 创建新科室

#### 3.3 编辑科室
- **路径**: `/admin/department/edit/{id}`
- **功能**: 修改科室信息

#### 3.4 删除科室
- **路径**: `/admin/department/delete/{id}`
- **功能**: 删除指定科室

### 4. 医生管理

#### 4.1 医生列表
- **路径**: `/admin/doctors`
- **功能**: 查看所有医生信息，包括：
  - 医生ID
  - 姓名
  - 科室
  - 职称
  - 专长

#### 4.2 添加医生
- **路径**: `/admin/doctor/create`
- **功能**: 关联用户和科室创建医生记录

#### 4.3 编辑医生
- **路径**: `/admin/doctor/edit/{id}`
- **功能**: 修改医生信息

#### 4.4 删除医生
- **路径**: `/admin/doctor/delete/{id}`
- **功能**: 删除指定医生

### 5. 预约管理

#### 5.1 预约列表
- **路径**: `/admin/appointments`
- **功能**: 查看所有预约信息，包括：
  - 预约ID
  - 患者姓名
  - 科室
  - 医生
  - 预约时间
  - 预约状态

### 6. 数据管理

#### 6.1 数据备份
- **路径**: `/admin/data/backup`
- **功能**: 执行数据备份
- **备份类型**:
  - 完整备份
  - 增量备份

#### 6.2 备份列表
- **路径**: `/admin/data`
- **功能**: 查看历史备份记录

#### 6.3 数据恢复
- **路径**: `/admin/data/restore/{id}`
- **功能**: 从备份恢复数据

#### 6.4 数据统计
- **路径**: `/admin/data`
- **功能**: 显示系统数据统计
  - 用户总数
  - 科室总数
  - 医生总数
  - 预约总数

### 7. 系统设置

#### 7.1 医院信息设置
- **路径**: `/admin/settings/hospital`
- **设置项**:
  - 医院名称
  - 医院地址
  - 联系电话
  - 医院简介

#### 7.2 预约设置
- **路径**: `/admin/settings/appointment`
- **设置项**:
  - 每天最大预约数
  - 预约提前天数
  - 号源释放时间

#### 7.3 系统配置
- **路径**: `/admin/settings/system`
- **设置项**:
  - 系统维护模式
  - 维护说明
  - 日志保留天数

## 管理员首页

### 仪表盘
- **路径**: `/admin/dashboard`
- **功能**: 显示系统概览信息
  - 用户统计
  - 科室统计
  - 医生统计
  - 预约统计

## 技术实现

### 控制器

1. **AdminController** - 核心管理员控制器
   - 用户管理
   - 科室管理
   - 医生管理
   - 预约管理

2. **PermissionController** - 权限管理控制器
   - 权限列表
   - 权限更新

3. **DataController** - 数据管理控制器
   - 数据备份
   - 数据恢复
   - 数据统计

4. **SettingsController** - 系统设置控制器
   - 医院信息设置
   - 预约设置
   - 系统配置

### 实体类

1. **User** - 用户实体
   - username - 用户名（唯一）
   - name - 姓名
   - role - 角色
   - gender - 性别
   - phone - 电话
   - email - 邮箱

2. **Department** - 科室实体
   - name - 科室名称
   - description - 科室描述

3. **Doctor** - 医生实体
   - user - 关联用户
   - department - 关联科室
   - title - 职称
   - specialty - 专长
   - schedule - 排班信息

### 服务层

1. **UserService** - 用户服务
   - 用户CRUD操作
   - 用户名唯一性检查
   - 密码加密

2. **DepartmentService** - 科室服务
   - 科室CRUD操作

3. **DoctorService** - 医生服务
   - 医生CRUD操作

4. **AppointmentService** - 预约服务
   - 预约CRUD操作

### 视图模板

1. **users.html** - 用户列表
2. **user-form.html** - 用户表单
3. **permissions.html** - 权限管理
4. **data.html** - 数据管理
5. **settings.html** - 系统设置
6. **departments.html** - 科室列表
7. **department-form.html** - 科室表单
8. **doctors.html** - 医生列表
9. **doctor-form.html** - 医生表单
10. **appointments.html** - 预约列表

## 安全配置

### 角色权限

- **ADMIN**: 可访问 `/admin/**`
- **DOCTOR**: 可访问 `/doctor/**`
- **NURSE**: 可访问 `/nurse/**`
- **PATIENT**: 可访问 `/patient/**`

### 认证方式

- 使用 Spring Security 进行认证
- 支持基于角色的访问控制（RBAC）
- 密码使用 BCrypt 加密存储

## 使用说明

### 登录管理员账户

1. 访问登录页面
2. 输入用户名: `admin`
3. 输入密码: `admin123`
4. 选择角色: 管理员
5. 点击登录

### 管理用户

1. 进入用户管理页面
2. 点击"添加用户"按钮
3. 填写用户信息
4. 选择用户角色
5. 保存用户信息

### 管理权限

1. 进入权限管理页面
2. 查看所有用户的角色
3. 修改用户角色权限

### 备份数据

1. 进入数据管理页面
2. 选择备份类型
3. 点击"开始备份"按钮

### 系统设置

1. 进入系统设置页面
2. 配置医院信息
3. 配置预约参数
4. 配置系统参数
5. 保存设置

## 注意事项

1. **密码安全**: 管理员密码应定期更换，避免使用简单密码
2. **数据备份**: 建议定期进行数据备份，防止数据丢失
3. **权限管理**: 谨慎分配用户权限，避免权限滥用
4. **日志记录**: 系统会记录所有管理操作，便于审计
5. **系统维护**: 进行系统维护时，建议提前通知用户
